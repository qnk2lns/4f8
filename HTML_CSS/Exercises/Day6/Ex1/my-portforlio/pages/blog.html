Viết HTML semantic giúp code dễ maintain hơn như thế nào?

Giới thiệu

Khi mới học lập trình web, nhiều người thường chỉ quan tâm đến việc giao diện có hiển thị đúng hay không. Tuy nhiên, khi project lớn dần, cách viết HTML sẽ ảnh hưởng rất nhiều đến khả năng đọc hiểu, bảo trì và mở rộng source code.

Một trang web không chỉ cần đẹp, mà còn cần có cấu trúc rõ ràng. Đây là lý do semantic HTML trở thành nền tảng quan trọng với developer Frontend. Khi sử dụng đúng các thẻ như <header>, <main>, <section> và <article>, code sẽ dễ hiểu hơn cho cả người đọc lẫn trình duyệt.

Semantic HTML là gì?

Semantic HTML là cách dùng thẻ HTML dựa trên ý nghĩa của nội dung, không chỉ dựa trên mục đích styling. Ví dụ: phần điều hướng nên dùng <nav>, nội dung chính nên nằm trong <main>, còn một bài viết độc lập nên dùng <article>.

Khi code semantic tốt, người khác có thể nhìn vào HTML và hiểu ngay cấu trúc trang. Điều này đặc biệt quan trọng khi làm việc nhóm hoặc quay lại sửa project sau một thời gian dài. Thay vì chỉ có nhiều thẻ <div>, bạn sẽ có một outline rõ ràng và dễ scan hơn.

Một lợi ích khác là accessibility. Screen reader và các công cụ hỗ trợ có thể hiểu tốt hơn nếu trang được chia bằng landmark đúng. HTML semantic tốt là bước đầu tiên để tạo website dễ tiếp cận hơn.

Thói quen code hiệu quả

Ngoài semantic HTML, developer cũng nên hình thành thói quen làm việc nhất quán. Ví dụ, sau khi chỉnh sửa file, tôi thường dùng Ctrl + S để lưu nhanh, sau đó kiểm tra giao diện trên trình duyệt.

Khi chạy thử code JavaScript, output có thể được in ra console. Ví dụ kết quả: Form submitted successfully. Những phản hồi nhỏ như vậy giúp debug nhanh và kiểm tra logic dễ hơn.

Ví dụ mã

Ví dụ đầu tiên là một cấu trúc HTML semantic đơn giản cho một trang web cá nhân. 

<!-- HTML semantic layout example -->
<header class="app-header">
    <nav class="app-header__nav">
        <!-- Main navigation links -->
    </nav>
</header>

<main class="app-main">
    <h1>About me</h1>

    <section class="about-profile">
        <h2>Về tôi</h2>
        <p>Short introduction content...</p>
    </section>
</main>

Ví dụ thứ hai là cách đặt tên class theo BEM để tránh class quá chung chung và khó maintain. 

/* BEM naming example */
.project-card {
    /* Block style */
}

.project-card__title {
    /* Element style */
}

.project-card__button {
    /* Element style */
}

.project-card--featured {
    /* Modifier style */
}

Ví dụ thứ ba là một đoạn JavaScript nhỏ để kiểm tra form submit. 

// JavaScript form submit example
const contactForm = document.querySelector(".contact-form");

contactForm.addEventListener("submit", function (event) {
    event.preventDefault();

    // Show success message after submit
    console.log("Form submitted successfully");
});
